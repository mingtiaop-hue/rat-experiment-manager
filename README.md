# 🐀 动物实验记录与数据管理系统 v3.4

糖尿病大鼠感染创面愈合实验 — 每鼠4伤口 | 拍照上传 | 愈合率 | GraphPad导出

## 快速开始

```bash
pip install streamlit pandas openpyxl Pillow qrcode[pil] plotly
streamlit run app.py
```

浏览器打开 `http://localhost:8501`

> **桌面一键启动**：双击 `AnimalExperiment`（桌面快捷方式），自动打开浏览器。

## 实验设计

- **不电刺激鼠** (8只): 7, 9, 10, 11, 13, 15, 16, 17
  - W1→Control / W2→SA / W3, W4→HJ-SA
- **电刺激鼠** (9只): 1, 2, 3, 4, 5, 6, 8, 12, 14
  - W1, W2→ES / W3, W4→DCC-HJ-ES

每鼠4个伤口，共68个伤口。伤口面积可用 ImageJ 后补计算。

## 功能

| 模块 | 说明 |
|------|------|
| 📋 日常录入 | 选天→进度条追踪→填面积→拖拽上传照片，支持删除重传 |
| 📦 批量上传 | 一键拖入4张照片，按 W1-W4 自动分配 |
| 🖼️ 照片长廊 | 单伤口 Day 1-14 全时间线，支持删除单张照片 |
| 🔬 跨鼠对比 | 同伤口位多鼠横向对比，只显示已上传的天，自动排序 |
| 📊 数据导出 | Excel 含原始数据 + GraphPad Prism 格式 + 鼠信息（含死亡类型） |
| 🧪 样本管理 | 取材日组织样本跟踪 |
| 🐁 状态总览 | 全部鼠/伤口状态，区分实验死亡💀 vs 取材处死🔵 |

## v3.4 新特性

- **录入进度追踪**：日常录入页顶部显示当日完成进度，列出待录入鼠
- **死亡类型区分**：实验死亡（意外/感染）vs 取材处死（按计划），不同图标
- **死亡天筛选**：鼠死后在后续天数自动隐藏，死亡当天仍可录入
- **照片删除**：日常录入和照片长廊均可删除误传照片
- **全新侧边栏 UI**：彩色进度条 + 分组状态条 + 核心指标卡片
- **对比页优化**：只显示有照片的天数，杜绝空白占位

## 手机端

PC 启动后侧边栏显示二维码，手机扫码即可拍照上传。照片自动压缩（1920px, JPEG 80%）。

## 数据存储

所有数据存于 `experiment_data/`：
- `experiment.db` — SQLite 数据库
- `photos/` — 压缩后的照片（按分组/伤口ID 分目录）
- `exports/` — Excel 导出文件 + 数据库备份
